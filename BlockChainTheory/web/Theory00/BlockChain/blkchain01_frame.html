Transaction
We can define a transaction as:

APPLY(Status,Transaction) -> Status' or Error

Where the applicatiom of Transaction to a Status can result in a new status Status' or in Error

In our the banking example described below where Alice gives a check to Lucy.

APPLY({ Alice: $50, Lucy: $50 },"send $20 from Alice to Lucy") = { Alice: $30, Lucy: $70 }
or:
APPLY({ Alice: $50, Lucy: $50 },"send $70 from Alice to Lucy") = ERROR

Transaction validation, recording and ledger (libro mastro) are mantained by a central authority.
