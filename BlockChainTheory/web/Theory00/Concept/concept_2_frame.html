Block Chain Concepts

A network of peer-to-peer nodes.

The network nodes imply that a certain peer-to-peer network protocol is used.

There are basically 2 type of blockchain networks:


  • Permissionless Anyone can join the network hence partecipating to the transaction validation. (Bitcoin and Ethereum).
  • Permissioned Access to the network (for nodes and partecipants) is restricted to a set of known well known entities.

An ongoing debate disputes whether a private system with verifiers tasked and authorized (permissioned) by a central authority, should still be considered a block chain.

Proponents of permissioned or private chains argue that the term “blockchain” is rightly applied to any data structure which batches data into blocks which are timestamped and that these blockchains serve as a distributed version of multiversion concurrency control (MVCC) in databases.

The opponents say that the permissioned systems look like traditional corporate databases, not supporting decentralized verification of the data, and that such systems are not hardened against tampering and revision by their operators.
